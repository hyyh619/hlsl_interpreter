# Step 214 — PlanetCoaster o4.xy 次级 UV 深挖：发散锁定到派生 t2 索引，受限于 capture 无中间量

Date: 2026-07-21

## Prompts（本步任务）

修复 PlanetCoaster ×29 逐顶点数据发散（以 event10418 为例，o4=TEXCOORD4 从 row6 起偏）。

## 结论先行

把 event10418 追到**最底层**：发散**仅在 o4.xy（次级 UV 集）**——sv_position、o0–o3、**o4.zw（另一 UV
集）全部逐位正确**。o4.xy 的错源是**派生出的 t2 索引落到了错误元素**：v6 我方读 `t2[1].val[0]=0xb5cffe09`
（低16=65033），而 golden 隐含需要 `0x80CB8141`（低16=33089、高16=32971）——该值**不在**我方索引到的
t2 邻近元素里。索引由 6+ 层打包浮点读链派生，其正确中间量**不在 capture 里**（golden 只存最终 o4），
无法在不引入投机改动的前提下反推。为**保 190/190 回归全绿**，本步**不改代码**，只落最深定位。

## 执行 · 逐层追踪（v5 通过 vs v6 失败）

draw：`DrawInstanced, InstanceOffset=50, TriangleStrip, 2688 verts`；v0–63 同属一个 meshlet
（`r0.y = (v1>>6)+v0.x` 对 v1<64 恒定），逐顶点差异仅来自 v1 低位驱动的打包索引。

1. **失败面极窄**：row6 起**只有 `TEXCOORD4[0]/[1]`（=o4.xy）失败**，sv_position / o0–o3 / o4.zw 全过。
   → 顶点索引、几何拉取、蒙皮、切线帧、主 UV 全对；仅 o4.xy 这一支的解码错。

2. **o4.xy 解码链**：`o4.xy = 16*r1.yz`，`r1.yz = r2.xy/32768 - 1`，`r2.xy = asint(r0.y)&0xffff / >>16`，
   `r0.y = 三元选择(t2[idx].val 组件)`，`idx = flag_r0.y + (r0.x>>2)`。

3. **排除项**（逐一验证）：
   - **自别名三元并行语义**：`r1.yw = r1.yy ? r3.zw : r3.xy`（条件寄存器 r1.y 同时是目的）——独立测试
     确认**条件按向量先读后写、并行语义正确**，非 step-205 类 movc 竞态。
   - **r0.x 打包值正确**：o4.zw 与 sv_position 同样依赖 r0.x 且全过 → r0.x 无误。
   - **t2 读字节正确**：`t2[1].val[0]` 逐字节 = 0xb5cffe09，与缓冲一致 → 是**索引**错，不是解码错。
   - **denormal-as-index 自洽**：`t0[r1.w]` 中 r1.w 显示 "0.0000" 实为非规格化浮点，经 step-212 的
     `_eval_subscript`（<2⁻³⁰ 按 asuint）读 `t0[大索引]` 的真实数据；t0[0] 实为全零，印证索引走 asuint。

4. **卡点**：golden 需要的 `r0.y=0x80CB8141` 不在我方 block1 索引（=1）到的 t2 元素中；正确 t2 索引由
   `flag_r0.y`（t0 大索引的 val[4]）+ `r0.x>>2` 派生，而**该 flag 的正确中间值无法从 capture 得到**
   （只有最终 o4 是 golden）。block2 用相邻的 `flag_r0.z`（val[5]）却正确——两支索引数学同构，差异仅在
   读到的 flag 值，属**数据相关**，无同构可循的通用修复点。

## 结果

- **未改代码**；回归维持 **190/190 全绿**；Dump 仍 55。
- 把 PlanetCoaster 发散从"逐顶点数学发散"**收窄到"o4.xy 次级 UV 的派生 t2 索引落错元素"**，并逐一排除了
  三元并行语义 / r0.x / t2 字节解码 / denormal 索引四类可疑点。
- **硬约束**：进一步定位需要 GPU 侧中间量（如 golden 的 t2 索引 / 各级 flag），而 capture 仅存最终
  VS 输出 → 在现有数据下无法反推正确索引，投机改动风险高且会威胁绿线。

## 关键结论

- 该簇失败**不是通用 bug**，而是**多级打包索引在某一级读到与 GPU 不一致的派生值**；本步已排除全部可通用化的
  嫌疑（并行语义、r0.x、字节解码、denormal 索引），剩余分歧是**数据相关且缺中间量参照**。
- 工程判断：无法以低风险通用修复推进时**不冒险已绿的 190 回归**；后续若要继续，唯一可行路径是**重新
  capture 带各级中间寄存器 dump**（或接入 DXBC 参考执行），据此逐例对齐 t2 索引。
