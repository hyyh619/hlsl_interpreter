# Step 215 — manhattan ×5 深挖：粒子出生哈希 frac(sin(大数)·big) 的 GPU float32 精度极限

Date: 2026-07-21

## Prompts（本步任务）

修复 manhattan ×5（每例仅 1–2 顶点失败，998~999/1000），用户判断为"退化哨兵顶点的 float 精度"。

## 结论先行

深挖 event124 row411：失败顶点是**粒子出生（birth）粒子**，其位置来自伪随机出生哈希
`frac(sin(78.233·(635+vid))·87362)`。vid=411 → sin 参数 ≈ **81831.7**（大数）。GPU 的 float32 sin 在该
参数下 ≈ **−0.46685**，与 libm/float64（−0.466794）及全部 4 个约简模型（libm/f32red/cw2/cw2fma）都差
约 **5×10⁻⁵**；而 `frac(sin·87362)` 把这 5e-5 放大 **87362 倍** → 完全不同的伪随机值（golden 0.73438
vs 我方 0.384）。这是 **GPU 超越函数（sin）在大参数下的可复现性极限**，非逻辑错——与用户"float 精度"
判断一致。**无法在不知道该 GPU 具体 sin 实现的前提下复现**。为保 190/190 全绿，本步**不改代码**。

## 执行 · 追踪与数值验证

### 1. 澄清：`(10000,10000,10000)` 是**输出哨兵**，非输入

shader（`VS_shader.hlsl` 经 dedup 别名 = `GS_shader.hlsl`，粒子发射器）末尾
`o0.xyz = alive ? pos : float3(10000,10000,10000)` —— 10000 是**死亡粒子的输出哨兵**。上一步"输入哨兵"
的表述有误：row411 实为**存活的 birth 粒子**，走 `if (r0.x != 0)` 分支。

### 2. 定位到出生哈希

birth 分支：
```
r0.x = 635 + vid;                              # 1046
r0.xyzw = float4(78.233,12.99,78.233,53.36) * r0.xxxx;   # ≈ [81831.7, 13587, 81831.7, 55817]
r0.xyzw = sin(r0.xyzw);
r0.xyzw = float4(87362,43758,87362,12234) * r0.xyzw;
r0.xyzw = frac(r0.xyzw);                        # 伪随机 [0,1)
r1.xyz = r0.yzw*2 - 1;  o3.xyz = r1.xyz;        # 随机方向 → o3=TEX_COORD3
```
golden o3 = [−0.42334, 0.46875, 0.09961] → r0.yzw = [0.28833, **0.73438**, 0.54980]；
我方 = [0.4048, **0.3828**, 0.5049]。哈希本身发散。

### 3. 数值验证（sin@81831.7）

| 计算方式 | sin | frac(sin·87362) |
|---|---|---|
| f64 libm（arg f64） | −0.466794 | 0.80167 |
| libm（arg f32 舍入，=我方现状） | −0.467703 | 0.38432 |
| f32red / cw2 / cw2fma | 均不匹配 | 0.53/0.80/… |
| **GPU（由 golden 反推）** | **≈ −0.46685** | **0.73438** |

GPU 的 sin 与全部模型差 ~5e-5；frac 放大 87362× → 差 ~0.07~0.9。**4 个约简模型无一命中**（实跑 4 种
trig_model，manhattan 均 999/1000 不变）。

### 4. 为何仅个别顶点失败

大多数顶点走 else 分支（无大数 sin），或其 birth 哈希的 sin 差异恰好使 frac 落在容差内（frac 需 sin
匹配到 0.005/87362 ≈ **5.7e-8** 才过——这是 f32 epsilon 量级）。row411 的大参数 sin 差异超过该阈值 →
唯独它 fail。属**精度彩票**，非特定逻辑缺陷。

## 结果

- **未改代码**；回归维持 **190/190 全绿**；Dump 仍 55。
- 用数值证据**证实**（非猜测）manhattan 失败 = 出生哈希 `frac(sin(大参数)·big)` 的 GPU float32 sin
  可复现性极限；已实跑排除全部 4 个 trig 约简模型。

## 关键结论

- 该类失败**非逻辑 bug、非可通用修复**：伪随机哈希对 sin 的要求高达 ~5.7e-8，而 GPU 大参数 float32 sin
  与 libm/所有约简模型差 ~5e-5，frac 放大后必然发散。
- 需该具体 GPU（manhattan=GFXBench 移动 GPU）的 sin 精确实现方能复现；capture 只有一个数据点，无法反推。
- 工程判断：不以投机改动（如新增拟合某 GPU 的 sin 表）冒险已绿的 190 回归。
