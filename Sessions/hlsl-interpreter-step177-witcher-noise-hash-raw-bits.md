## 21979 定案：整数噪声哈希的三层语义缺失（修复后 840/840 全 PASS）

shader 是纯程序化草地摆动：时间驱动的**整数格点哈希噪声**
（bfrev + xorshift + imul/imad 二次哈希 + smoothstep 插值），无纹理。
复现器（replica21979.py）通过逐 disasm 指令对照，以 golden TEXCOORD2
（世界坐标，直接暴露噪声值）为 oracle，**840/840 比特级复现**后确证 GPU 语义：

1. **`iadd`/`bfrev` 直接消费 round_ni 结果寄存器的原始 float32 位**
   （disasm 无任何 ftoi/ftou！）——3Dmigoto 渲染成 `(int)x`/`(uint)x`
   的强转实为**位记号**。floor(−3345.0) 参与 iadd 的是 0xC5511000，不是 −3345。
2. **`ishr` 是算术右移**，而 3Dmigoto 把 ishr 和 ushr **都**渲染成
   `(uint)x >> n`——文本层无法区分（22229 的 ubfe 展开需要逻辑移位，
   同样的文本！），只能 disasm 感知修复。
3. imul/imad 按 32 位回绕（Python bigint 需 wrap）。

### 解释器修复（hlsl_interpreter.py + render.py）

1. **`reversebits`（bfrev）实现**：取参数（剥掉 int-cast 记号）的 float32
   位模式、32 位反转，返回 `_RawBits`（int 子类，标记"寄存器位模式"）。
   此前整条语句被静默跳过、寄存器留旧值。
2. **`_RawBits` 伙伴取位规则**（binary_op 的 +,−,*）：一侧是 _RawBits 时，
   另一侧的 float 伙伴按位重解释（DXBC 整型指令吃原始位；反编译的 cast
   标记因**贪婪解析**——`(int)a+(int)b` 解析成 `(int)(a+(int)b)`——不可靠，
   凡 float 皆取位）；运算内联做 32 位回绕，**绕过 execute_binary_op 的
   f32_emulation 舍入**（否则位模式被转成 float、链条断裂）。
3. **`fix_shift_signedness(code, disasm)`**（render.py 接线）：HLSL 的 `>>`
   与 disasm 的 ishr/ushr 按程序序 1:1 配对（数目不符则整体不动——ubfe
   展开天然错开而自我保护），ishr 实例的 `(uint)` 改写为 `(int)`；
   evaluator 中 `(int)x >> n` 走算术（符号扩展）路径。
4. **`_eval_bitwise_operand` 放宽**：位操作上下文中 int-cast 的 float 操作数
   一律位重解释（原先只对顶点输入放行）。
5. **`_apply_swizzle_assign` 整数保型**：标量赋值原先强转 float——
   `_RawBits(559779)` 存成 559779.0，读回 `(int)` 时被(4)位转成 float 位
   模式，全链崩坏。现在 int（含 _RawBits）原样存储。
6. **ftou 负数→0**（D3D 规范）：`(uint)` 强转 finite 负 float 值时返回 0
   （denormal 位模式优先级更高，不受影响）。

### 首轮回归暴露的误伤（已修）

放宽 `_eval_bitwise_operand` 后 5 个 Octopath 案清零（2135/2912/3012/1357/1828）：
`(uint)1 & ~bitmask` 里的字面量 `1` 在解释器中求值为 float 1.0，被新规则位转成
0x3F800000。修复：**数字字面量节点不取位**（`_is_numeric_literal_node` 守卫）——
字面量是真值，只有寄存器/变量读才可能是位模式。修后 5 案全恢复、21979 保持 840/840。
