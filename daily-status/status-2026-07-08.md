## Interpreter fixes worth noting

The most reusable, non-case-specific fixes from this batch:

- **Ternary precedence in the expression parser (step 201).** `(int)cond ? A : B` was being parsed as `(int)(cond ? A : B)`, so the cast swallowed the whole ternary and truncated the selected branch to 0 — collapsing tangent-basis vectors to zero. The ternary conditional (lowest precedence in HLSL) is now split before the cast branch. This is a core parser correctness fix and the highest-risk-if-wrong of the batch.
- **B8G8R8A8 R/B channel swap (step 202).** UNORM 1-byte-per-component vertex attributes with a `B8G8R8*` format are stored in memory as `[B,G,R,A]` but must reach the shader as `.x=R, .y=G, .z=B`. `_decode_vertex_element` now swaps components 0 and 2. General to any B8G8R8A8/B8G8R8X8 vertex attribute.
- **asfloat / bit-select through the FMA path (steps 196/198).** Reinterpreted bit patterns (`asfloat(exp<<23)`, `&`-masked selects) are now carried through the fused-multiply-add path as raw bits instead of being coerced to integers.
